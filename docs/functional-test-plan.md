# MVP + V1.1 功能测试矩阵

## 自动化功能测试

后端自动化测试入口：

```powershell
cd backend
.\.venv\Scripts\python.exe -m pip install -r requirements-dev.txt
.\.venv\Scripts\python.exe -m pytest tests -q
```

| 编号 | 功能 | 覆盖点 |
| --- | --- | --- |
| API-01 | 模型列表 | `/api/models` 代理 Ollama `/api/tags`，配置默认模型缺失时回退到可用模型 |
| API-02 | Ollama 诊断 | `/api/health/ollama` 返回规范化 base URL、tags URL 和模型列表 |
| API-03 | 流式聊天 | 用户消息保存、助手流式内容返回、助手消息写入历史、首条消息生成标题 |
| API-04 | 文本附件 | Markdown/TXT 上传、解析、绑定消息、解析文本进入发给模型的上下文 |
| API-05 | 视觉附件 | 图片上传、存储、Base64 编码、视觉模型 payload 包含 Ollama `images` |
| API-06 | 损坏附件 | 损坏 PDF 上传后返回 `failed` 状态，不把接口打成 500 |
| API-07 | 附件隔离 | 一个会话不能把另一个会话的附件 ID 用于聊天 |
| API-08 | Token 认证 | 认证开启时保护会话接口，错误 Token 返回 401，配置和 Ollama 状态接口保持公开 |
| API-09 | 快捷提示词 | 自定义模板创建、读取、更新、启用/禁用和删除；内置模板由启动种子保证 |
| API-10 | 附件解析详情 | 详情返回解析预览、错误、截断标记、原始字符数、实际字符数；文件可重新解析 |
| API-11 | 会话搜索 | 标题、消息内容、附件文件名三类匹配都能返回对应会话 |
| API-12 | 诊断配置 | `/api/health/config` 返回 Ollama 地址、默认模型、文件截断上限和认证开关 |
| API-13 | 消息重试 | 对已有用户消息重试时新增助手回答，不重复保存用户提问 |

## 手工/浏览器功能测试

| 编号 | 场景 | 操作 | 预期 |
| --- | --- | --- | --- |
| UI-01 | 页面启动 | 打开前端 | 模型列表加载，默认模型可选择 |
| UI-02 | 文本发送 | 输入问题后按 `Enter` | 发送消息并流式展示回答 |
| UI-03 | 换行 | 输入框按 `Shift+Enter` | 输入框插入换行，不发送 |
| UI-04 | 历史恢复 | 刷新页面并点击历史会话 | 消息和附件恢复 |
| UI-05 | Markdown | 让模型输出表格和代码块 | 表格渲染，代码复制按钮可用 |
| UI-06 | 粘贴图片 | 截图后 `Ctrl+V` | 待发送区显示缩略图，发送后历史保留 |
| UI-07 | 拖拽文件 | 拖入 PDF/Markdown | 显示文件卡片，回答可基于文件 |
| UI-08 | 长消息滚动 | 打开长回答会话 | 聊天窗口可上下滚动，输入框保持在底部 |
| UI-09 | Token 登录 | `AUTH_ENABLED=true` 后首次打开 Chrome | 登录页显示，正确 Token 进入聊天，刷新仍保留 |
| UI-10 | 快捷提示词 | 点击内置模板按钮 | 模板内容写入输入框但不自动发送，可继续编辑 |
| UI-10A | 模板后粘贴发送 | 点击快捷提示词后粘贴图片/文件，再按 `Enter` | 附件进入待发送区，焦点回到输入框并发送，不重复插入模板 |
| UI-11 | 模板管理 | 新增、编辑、禁用、删除模板后刷新 | 模板状态与自定义内容保留 |
| UI-12 | 解析预览 | 点击历史消息里的文件附件 | 文件详情面板展示解析文本、状态、字符数、截断和错误，可重新解析 |
| UI-13 | 搜索恢复 | 分别按标题、消息内容、附件名搜索 | 点击结果打开对应历史会话 |
| UI-14 | 诊断页 | 打开“Ollama 诊断” | 地址、配置、本地模型、tags 测试和 chat 测试结果可见 |
| UI-15 | 极矮窗口首屏 | 新建会话后把浏览器内容区压到约 `2048x224` | 快捷提示词栏可横向滚动，输入框与发送按钮仍完整可用 |
| UI-16 | 消息重试 | Ollama 故障恢复后点击用户消息上的重试按钮 | 重新流式生成回答，历史中不会复制一条相同提问 |

## 真实 Ollama 冒烟测试

推荐使用本机已安装模型跑一轮：

```powershell
Invoke-RestMethod http://127.0.0.1:8000/api/health/ollama | ConvertTo-Json -Depth 8
Invoke-RestMethod http://127.0.0.1:8000/api/models | ConvertTo-Json -Depth 8
```

对支持图片的模型再补测：

1. 发送一条纯文本问题。
2. 上传一份 Markdown 或 PDF 并让模型回答文件内可验证事实。
3. 上传一张内容明确的图片并让模型回答颜色、文字或对象。

## 回归检查顺序

1. 先跑后端自动化测试和前端构建。
2. 用真实 Ollama 检查 `/api/health/ollama`，确认 `tags_url` 没有出现 `/api/api/tags`。
3. 用浏览器分别跑一轮文本聊天、历史恢复、文件上传预览、图片上传、搜索、诊断页。
4. 切到低高度窗口重看新建会话，确认输入框没有被快捷提示词栏挤掉。
